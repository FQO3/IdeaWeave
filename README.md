## 🧠 项目简介

**灵织（IdeaWeave）** 是一款面向个人的「灵感全生命周期管理」工具，尝试把分散的灵感从 **“点”** 一路编织到可执行的 **“块”**：

- **点（Point）**：极低阻力的灵感记录（文字 / 图片 / 音频 / 视频等）
- **线（Line）**：AI 自动标签与关联，生成「灵感星图」
- **面（Plane）**：基于双链笔记与无边白板的头脑风暴与协作（规划中）
- **块（Block）**：AI 协助做任务拆解 / 优先级 / 日程规划，并引导用户去真正适合执行的工具（规划中）

与传统「笔记工具」或「项目管理工具」不同，灵织的定位是：  
> **填补“灵感产生”和“项目执行”之间的断层，做一个轻量的灵感孵化器，而不是全能工作台。**

当前仓库为 **MVP 最小可用版本**，核心聚焦在：

- 灵感点的快速记录与 AI 异步分析  
- 点与点之间的自动关联  
- 可交互的「灵感星图」可视化

## ✨ 核心特性一览

### 1. 灵感「点」：多模态输入与 AI 异步分析

- 支持多种形式记录灵感：
  - 文本
  - 图片（调用对应多模态模型做摘要）
  - 音频（语音转写后做摘要）
  - 视频（规划中）
- **AI 异步分析，无需用户等待**：
  - 灵感一保存即成功，不阻塞后续记录
  - 后台异步调用大模型，对笔记进行：
    - 类型识别（灵感 / 规划 / Todo）
    - 标签生成
    - 摘要生成

并通过「少量多次调用」的策略，降低单次 Prompt 的上下文长度，从而节约 token 开销。

---

### 2. 「线」：AI 生成的灵感星图与关联关系

- 引入 **标签化分类查找 + 文本匹配度计算**：
  - 先根据标签匹配度（例如 ≥60%）筛选候选笔记
  - 再交给 AI 计算具体关联与原因
- AI 为每条边生成：
  - 关联度指数（0–1）
  - 关联理由文本

在 UI 层使用自主编写的拓扑绘图引擎，实现：

- 灵感星图（Force-directed Graph）
- 节点可拖拽、缩放
- 按关联度筛选与过滤弱连接

> 星图不是“装饰页面”，而是一个真正可操作的工作视图。

---

### 3. 自研星图绘制引擎（Topology Engine）

为了让灵感网络真正「看得懂、用得上」，本项目实现了一个自研的星图引擎（近 2k 行 TS 代码）：

- 物理模拟：
  - 引力 / 斥力 / 惯性 / 阻力
- 完整交互：
  - 节点拖动 / 自动布局
  - 连线 Hover / 点击查看「关联理由」
  - 节点弹窗编辑（标题、内容、标签等）
  - 项目标签属性修改
  - 星图统计信息查看
- **局部更新**：
  - 编辑某个节点或边后，只局部重新计算/渲染
  - 避免整张图重绘，保持用户对布局的空间记忆

---

### 4. 双链笔记与无边白板（规划中）

未来计划：

- 引入 **双链笔记**：
  - 支持在任意文本上创建引用指向其他笔记或白板
  - 实现真正意义上的「文字级引用」
- 开发 **Brain Storm 无边白板**：
  - 以「项目」为单位创建白板
  - 自由绘制 + 笔记节点
  - 多人实时协作（团队 + 灵感社区）
- AI 在白板中的角色：
  - 作为 Assistant 提供广度优先的发散建议
  - 利用社区大数据做灵感提示，而不是替代人的思考

---

### 5. 「块」：从灵感网络到可执行计划（规划中）

- 引入 Todo / 任务列表视图：
  - AI 综合用户灵感内容与日常习惯，自动生成任务建议
- 提供多种规划范式：
  - 四象限优先级（重要/紧急）
  - 日程表 / 甘特图等
- 关键理念：
  - **引导用户离开应用，到真正的执行环境中去**
  - 通过多种导出方式接入外部工具，而不是试图“什么都自己做”

---

### 6. 导出与“退场哲学”（规划中）

灵织不会试图把用户「锁」在系统内，而是：

- 支持多格式导出：
  - Markdown 文档
  - 任务列表（支持简单标准化 schema）
  - API 接口（供其他工具接入）
- 对接外部常用工具：
  - 任务/项目管理工具
  - 文档/笔记工具
  - 团队协作平台

> 产品理念：**“做到块之后，把人推向真实世界与合适工具。”**

## 🧪 设计理念与边界

### AI 到哪一步为止？

- **AI 只作为辅助与「粘合剂」**：
  - 串联「记录 → 整理 → 关联 → 规划」各阶段
  - 不直接生产用户的核心灵感内容  
- **避免“模板化方案同质化”**：
  - 更强调提供多种结构化视角和思考路径
  - 提示关键问题，而不是给唯一答案
  - 决策与最终判断始终交给用户本人  

---

### 隐私与知识产权（规划中）

在未来的「灵感社区」与协作场景中，将重点思考：

- **灵感链路开放范围**：
  - 完全公开
  - 半公开（对部分人或匿名开放）
  - 仅对信任关系可见
  - 定时公开 / 渐进式公开策略  

- **灵感协议设计（类似软件 License）**：
  - 是否允许 fork / 二创
  - 如何要求标注来源
  - 如何处理多人协作创作的归属与署名
  - 针对企业 / 团队使用的额外协议与边界
